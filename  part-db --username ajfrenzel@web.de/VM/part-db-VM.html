PartDbVM

Version dok-0.1

Benötigte Komponenten

  • Vmware Server 2.0

  • part-db.tar.bz2

  • Für Windows: puTTY und WinSCP



Installation des Vmware Server 2.0

Das Paket kann kostenlos von http://downloads.vmware.com/d/info/datacenter_downloads/vmware_server/2_0 heruntergeladen werden. Zum Betrieb ist eine Seriennummer notwendig, die ebenfalls kostenlos bezogen werden kann.

Die Administration des Servers erfolgt über ein Browser-Plugin, das evtl. separat installiert werden muß.

PartDbVM in den Vmware Server integrieren

  • Als erstes wird part-db.tar.bz2 in das Verzeichnis für virtuelle Maschinen des Servers entpackt.
    Unter Windows kann dafür 7zip verwendet werden.

  • Starten des Vmware Server Administrationsplugins im Browser

  • Unter Virtual Machine|Add Virtual Machine to Inventory unter <server>|Standard|part-db|part-db.vmx auswählen.

Starten der virtuellen Maschine

  • Im Navigationsbereich links part-db anklicken.

  • Dann im mittleren Bereich den Reiter Console auswählen und in das schwarze Fenster klicken.
    Die VM startet.

  • Dann nochmals in das schwarze Fenster klicken und warten, bis sich das Konsolfenster der VM öffnet.

  • Wenn die VM hochgelaufen ist, erscheint eine blaue Box, in der die IP-Adresse der VM angezeigt wird.

Die PartDB-VM ist jetzt im LAN per Browser aufrufbar.

Konfigurieren der IP-Adresse

Am unteren Rand der Box ist ein Link auf das Advanced Menu. Man erreicht es, indem man in das Konsolfenster der VM klickt und dann Return eingibt.

Es ist ratsam, der VM eine statische IP-Adresse zu geben und die in /etc/hosts auf Linux bzw. in c:\windows\system32\drivers\etc\hosts auf Windows einzutragen.

Die Navigation in den folgenden Menüs geht über die Pfeiltasten im numerischen Tastenfeld.

Den Cursor bekommt man mit einem gleichzeitigen Druck auf Strg + Alt wieder frei.

Aufrufen der PartDB im Browser

  • Die URL der PartDB wird nach dem Muster http://<ip-addresse> in die URL-Zeile des Browsers eingegeben.

Administration der PartDbVM

Die PartDbVM ist in einer vorkonfigurierten TurnKey MySQL VM installiert. Weitere Informationen dazu sind hier: http://www.turnkeylinux.org/mysql zu finden.

Momentan besteht noch ein Zugriffskonflikt zwischen PartDB und PhpMyAdmin, der durch zwei Skripte unter /root aufgelöst wird.

Administration von Windows aus

Die bequemste Möglichkeit ist, die beiden Pakete puTTY und WinSCP zu installieren.

  • Dateitransfer kann sehr komfortabel mit WinSCP durchgeführt werden:
    + WinSCP starten
    + New klicken und die IP-Adresse des PartDB-Servers unter Host eintragen
    + Unter User wird     root eingetragen. Das Paßwort bleibt leer.
    +     Log in klicken.

  • Die Linux-Shell in der VM wird über puTTY bedient:
    + puTTY starten
    + IP-Adresse des Servers unter Host Name eintragen
    +     Connection type: ssh
    +     Open klicken
    + Es öffnet sich ein Fenster mit dem Text     login as: Eingabe: root
    + Anschließend befindet man sich im Home-Directory /root

Administration von Linux aus

  • Dateitransfer erfolgt über sftp root@<ip-adresse>

  • Die root-Shell wird über ssh root@<ip-adresse> erreicht

Administration der PartDbVM

Achtung: Weder root noch MySql haben Paßwörter. Die VM sollte so nicht auf dem Web zugänglich gemacht werden!

  • Man loggt sich (über ssh oder puTTY) als root in die VM ein.

  • Nach dem Login befindet man sich im Home-Directory /root

Dort liegen drei Skripte:

  • Der Aufruf ./myadmin aktiviert PhpMyAdmin.

  • Der Aufruf ./part-db aktiviert PartDB

  • Der Aufruf ./autoprice liest die Preise von Reichelt- und Pollin-Artikeln ein

Je nach dem, welches der Skripte myadmin oder part-db als letztes gelaufen ist, meldet sich beim Aufruf des Servers im Browser entweder PhpMyAdmin, oder PartDB.



Weitere Informationen über die zugrundeliegenden TurnKey VM gibt es hier: http://www.turnkeylinux.org/docs/ubuntu-docs

Bekannte Probleme

  • Die virtuelle Platte der PartDbVM wird als „Hard disk not valid“ angezeigt.
    Dieser Fehler ist Harmlos. Weitere Info hier: http://www.turnkeylinux.org/forum/support/20091102/vmware-server-2-hard-disk-not-valid
